Common Python Beginner Mistake (2) - return value 'None'

The following statements look very innocent. 
q = 123
...
q = l.append('b')
...

Beginner will only discover a problem when they try to use the variable ‘q’ later down the track. They will be surprised that the variable ‘q’, for some reasons, has been set to None. 

print q+1
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for +: 'NoneType' and 'int'

Most developers from static languages backgrounnd are used to have the compilers to catch this kind of programming error, namely, a function that returns nothing (‘void’) should not appear in the RHS of an assignment operand.

But it is not the case for python. If a function returns nothing, the variable on the LHS will be set to None

It is a very subtle problem. Beginners may have written the assignment due to mistake or oversight. Then they tend to spend a lot of time to trace down the ‘bug’
